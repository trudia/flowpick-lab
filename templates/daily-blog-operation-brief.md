# TrendFlow Daily Blog Operation Brief

TrendFlow는 실제 6시간을 내부 운영상 1 blog-day로 본다. 이 템플릿은 03:00, 09:00, 15:00, 21:00 KST 체크포인트에서 사용한다.

## 체크포인트 구분

- 03:00 / 15:00: 가벼운 점검. 큰 작업을 새로 벌리기보다 현재 계획이 여전히 맞는지, 작은 수정이 필요한지 본다.
- 09:00 / 21:00: 깊은 점검. 우선순위, 완료 작업, 막힌 지점, 다음 행동, 사용자 입력 필요 여부를 정리한다.
- 의미 있는 변경이 없으면 억지 제안을 만들지 않는다. `오늘은 별도 개선 제안 없음`이라고 말해도 된다.

## 오늘 점검

1. 홈, 최신 글, 카테고리, 검색, 아카이브, 사이트맵 중 깨진 동선이 있는가?
2. TrendFlow와 TopClass Coupang이 하나의 운영 흐름처럼 이어지는가?
3. 이미 끝난 작업을 다시 하거나, 한쪽 기준만 반영해 정보 불균형이 생기지 않았는가?
4. 핵심 영역 중 부족한 곳은 어디인가?
   - AI 입문 / AI 활용 윤리
   - 집중 회복
   - 공간 정리
   - 커리어 & 교육 트렌드
   - 콘텐츠 부업
   - 구매 기준 / 쇼핑 리뷰
5. 오늘 가장 효과적인 작업은 무엇인가?
   - 신규 정보성 글
   - 기존 글 보강
   - 내부 링크 개선
   - 카테고리 허브 개선
   - 검색엔진 제출 / 색인 확인
   - 쇼핑 글 후보 정리
   - UI / 문구 개선
   - 별도 작업 없음

## 트렌드 판단

트렌드는 단순히 지금 많이 보이는 주제가 아니다. 글이나 쇼핑 후보를 고르기 전에 아래를 확인한다.

1. 과거에 어떤 불편이나 변화가 반복되었는가?
2. 지금 영향을 주는 환경은 무엇인가? 계절, 날씨, 주거, 일 방식, AI/플랫폼 변화, 가격 부담, 사회적 관심을 본다.
3. 이 주제는 반짝 유행이 지나도 남는 기준이 있는가?
4. 누구에게 도움이 되고, 누가 굳이 하지 않아도 되는가?
5. `지금 꼭 필요하다`보다 `이런 상황이면 도움이 된다`로 설명하고 있는가?

## 실패 원인 점검

새 도구, 새 제품, 새 방법을 찾기 전에 왜 기대처럼 되지 않았는지 먼저 본다.

- 실행 환경이 맞았는가?
- 유지 비용이나 관리 부담이 과하지 않았는가?
- 사용 빈도와 보관 공간이 현실적인가?
- 후기, 인기순, AI 추천을 너무 믿은 것은 아닌가?
- 기대한 효과와 실제 문제가 어긋난 것은 아닌가?
