운영자 메모

파소나의 법칙은 글을 설계할 때만 참고하세요. 공개 글에는 공식 이름이나 P/A/S/O/N/A 같은 용어를 직접 쓰지 말고, 자연스러운 문장으로 바꿔 쓰세요.

v20부터는 네가 직접 건드릴 수 있는 영역을 분리했습니다. 디자인과 공개 HTML은 유지하되, 글/사진/링크의 원본 관리 파일을 따로 둡니다.

1. 글 내용 수정

content/posts.json 파일에서 제목, 요약, 이미지, 섹션 내용을 수정할 수 있습니다.

수정할 것파일설명
글 제목/요약content/posts.jsontitle, summary를 수정합니다.
글 요약content/posts.jsonimage 값을 assets/uploads/파일명.webp로 바꿉니다.
쿠팡 링크content/affiliate-products.jsonproduct_url의 수정예정을 실제 링크로 바꿉니다.

2. 사진 추가

  1. 이미지를 WebP 또는 JPG로 준비합니다.
  2. assets/uploads/ 폴더에 업로드합니다.
  3. content/posts.json의 image 값을 assets/uploads/my-image.webp처럼 바꿉니다.

3. 새 글 만들기

글 HTML 생성기를 열고 제목, 요약, 이미지, 본문을 입력하면 새 글 HTML을 만들 수 있습니다.

운영자 메모

는 글을 설계할 때만 참고하세요. 공개 글에는 , P/A/S/O/N/A 같은 용어를 직접 쓰지 말고 자연스러운 문장으로 바꿔 쓰세요.

문제 해결형의 법칙 적용법

새 글을 작성할 때는 아래 순서로 쓰면 광고처럼 보이지 않으면서도 행동 유도가 자연스러워집니다.

  1. P 문제: 방문자가 겪는 불편함을 먼저 말합니다.
  2. A 공감: 의지 부족이 아니라 환경이나 구조의 문제라고 설명합니다.
  3. S 해결: 먼저 바꿔야 할 기준을 제시합니다.
  4. O 제안: 이 글에서 무엇을 정리했는지 안내합니다.
  5. N 대상: 누구에게 적합한지 좁힙니다.
  6. A 행동: 관련 글 보기, 비교 박스 확인, 필요한 글 찾기 등 다음 행동을 제시합니다.

4. 지금 방식의 한계

GitHub Pages/Cloudflare 정적 사이트는 브라우저에서 직접 저장이 되지 않습니다. 그래서 지금은 “생성기에서 HTML 만들기 → GitHub에 파일 업로드” 방식이 가장 안전합니다.

운영자용 링크 수정 위치

방문자용 제휴 안내 페이지에는 운영자 문구를 넣지 마세요. 쿠팡파트너스 링크 수정은 아래 파일에서 관리하세요.

  • content/affiliate-products.json
  • data/affiliate-products.json
  • 각 글의 구매 박스 버튼 링크

전/후 이미지 운영 원칙

  • 전/후 변화형 글은 가능하면 한 장의 비교 이미지(정리 전 / 정리 후)를 함께 넣으세요.
  • 우선순위 비교 글은 좌우 비교 또는 2~3분할 비교 이미지로 이해를 돕는 것이 좋습니다.
  • 같은 이미지를 여러 글에 반복해서 쓰기보다, 글 주제에 맞는 비교 이미지를 따로 두는 편이 더 신뢰감을 줍니다.
